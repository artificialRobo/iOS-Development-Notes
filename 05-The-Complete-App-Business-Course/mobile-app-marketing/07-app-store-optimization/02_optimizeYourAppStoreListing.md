### Notes: Optimize your App Store Listing

## iOS App Store

* **Keyword field limit:** 100 characters.
* **Do not use spaces** between keywords—they waste character count.

  * Example: `coffee,bean,cafe` instead of `coffee, bean, cafe`.
* **No need to add plural forms or keyword combinations.**

  * The App Store automatically generates:

    * Plurals
    * Different keyword combinations
  * Example:

    * Instead of `coffee, coffee bean`
    * Use `coffee, bean`
* **Keywords cannot be changed instantly.**

  * Updating keywords requires publishing a new app update.
* **Choose keywords carefully** because they affect:

  * Search ranking
  * App downloads
  * Revenue

---

## Google Play Store

* Keywords are determined from:

  * **App name**
  * **App description**
* **Use target keywords naturally** throughout the description.

  * Avoid keyword stuffing (e.g., repeating "coffee" excessively).
* **Keyword placement matters.**

  * Keywords closer to the beginning of the app name and description receive more weight.
  * Example:

    * **"Coffee House"** ranks better for "coffee" than **"House Coffee"**.
* **Keep important keywords as close to the beginning ("top left") as possible.**

---

## Key Differences

| iOS App Store                                        | Google Play Store                                       |
| ---------------------------------------------------- | ------------------------------------------------------- |
| Uses a dedicated keyword field (100 characters).     | Uses app name and description for keyword indexing.     |
| No spaces between keywords.                          | Write keywords naturally in the description.            |
| Automatically creates plural forms and combinations. | No automatic keyword field; content determines ranking. |
| Keyword changes require an app update.               | Optimize by editing the app listing.                    |
