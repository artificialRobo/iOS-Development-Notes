# Step 2: Design

## Why App Design Matters

* A great idea and solid code are **not enough**.
* There are millions of apps on the App Store and Google Play, so good design helps your app stand out.
* Design creates an **emotional connection** with users.
* Users can uninstall an app in just a few taps.
* Around **50% of downloaded apps are deleted after their first use**, making first impressions critical.

---

## What Is Good Design?

Good design makes an app:

* Easy to use
* Visually appealing
* Consistent with platform standards

### Learn Platform Guidelines

Use official design resources:

**Apple**

* iOS Human Interface Guidelines (HIG)

**Google**

* Material Design Guidelines

These explain:

* Layouts
* Navigation
* Animations
* User interactions
* Platform-specific design principles

---

## Find Design Inspiration

Look at apps with excellent user interfaces.

Useful resources:

* Pttrns
* Mobile Patterns

These websites showcase UI examples such as:

* News feeds
* Login screens
* Settings pages
* Navigation menus

**Tip:** Be inspired by successful designs instead of copying them.

---

## Choose a Color Palette

A consistent color palette improves your app's appearance.

Recommended tools:

* Color Hunt
* Flat UI Colors (popular for iOS)
* Material Palette (for Android)

These tools provide professionally selected color combinations.

---

## Create a User Flow Diagram

A **user flow** maps the steps users take to complete a task.

Examples:

* Signing up
* Purchasing an item
* Creating a recipe

Benefits:

* Easy to sketch on paper
* Simple to modify before coding
* Helps identify usability issues early

---

## Create Wireframes

A **wireframe** is a simple, low-detail sketch of the app.

Characteristics:

* Usually drawn on paper
* Focuses on layout and navigation
* Doesn't include colors or graphics

Purpose:

* Plan screen structure before development.

---

## Build Interactive Prototypes

Use tools like **POP** to:

* Photograph wireframes
* Link screens together
* Simulate a working app

Benefits:

* Test navigation
* Gather feedback
* Improve usability before writing code

---

## Create Mockups

A **mockup** is a high-fidelity version of the app design.

Unlike wireframes, mockups include:

* Colors
* Icons
* Images
* Typography
* Final layout

Purpose:

* Show developers exactly how the app should look.
* Present ideas to clients or investors.

Popular design tools:

* Photoshop
* Illustrator
* Sketch
* Marvel

---

## When Mockups May Not Be Necessary

If you're both the **designer and developer**, you can often build the interface directly in:

* Xcode (iOS)
* Android Studio

This avoids creating the same design twice.

---

## Simple UI Design Tools

If you don't want to learn advanced design software, use drag-and-drop tools like:

* UXPin

These allow you to quickly assemble screens using ready-made UI components.

---

## Use Legal Design Assets

Avoid copying images directly from Google because of copyright.

Instead, use assets that are:

* Free for commercial use
* Licensed under Creative Commons (when appropriate)

Recommended resources:

* Icon Store (icons)
* Unsplash (high-quality photos)
* Resplash (free images)

---

## Design Before Coding

A common mistake is starting development immediately.

Problems:

* Changing screen layouts later requires rewriting code.
* Small design changes can become expensive and time-consuming.

Recommended workflow:

1. Validate the app idea
2. Create a user flow
3. Draw wireframes
4. Design mockups (if needed)
5. Build a prototype
6. Start development in Xcode or Android Studio

---

## Key Takeaways

* Design strongly influences user retention.
* Follow Apple's and Google's design guidelines.
* Study successful app interfaces for inspiration.
* Use professional color palettes.
* Plan user journeys with flow diagrams.
* Sketch wireframes before coding.
* Prototype ideas early to test usability.
* Create mockups for clear visual direction.
* Use copyright-free assets.
* Invest time in planning—it's much easier to change a sketch than rewrite code later.
