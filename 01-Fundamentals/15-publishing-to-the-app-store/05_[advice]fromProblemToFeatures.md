# Notes: From Problem to Features

## 1. Start with the Problem

Ask:

> What problem am I solving?

Example:

* Problem: Students forget assignment deadlines.

## 2. Define the Benefit/Outcome

Ask:

> What benefit does the user get?

Example:

* Never miss deadlines.
* Stay organized.
* Reduce stress.

## 3. Define How You'll Deliver That Benefit

Ask:

> What must happen for the user to get this benefit?

Example:

* User needs to record assignments.
* User needs reminders.
* User needs to see upcoming deadlines.

## 4. Convert Those into Features

Features are capabilities of the product.

Example:

* Add assignment feature
* Deadline calendar feature
* Notification/reminder feature
* Dashboard showing upcoming tasks

## 5. Break Features into Functionality

Functionality is the specific behavior inside a feature.

**Feature:** Add Assignment

Functionality:

* Enter title
* Enter due date
* Select subject
* Save assignment
* Edit assignment
* Delete assignment

---

Here's a simple framework:

```
Problem
   ↓
Benefit
   ↓
Solution
   ↓
Features
   ↓
Functionality
```

---

## Example: Food Delivery App

**Problem**

* People don't want to cook.

**Benefit**

* Get food delivered easily.

**Solution**

* Connect customers with restaurants and delivery drivers.

**Features**

* Browse restaurants
* Search food
* Order food
* Track delivery
* Payment system

**Functionality of "Order Food" Feature**

* Add item to cart
* Change quantity
* Remove item
* Apply coupon
* Confirm order

---

## Advice

When building a startup, don't start by listing 50 features.

Instead ask:

> "What's the smallest benefit I can deliver to one user?"

For the student planner app, maybe the MVP is just:

* Create assignment
* Set deadline
* Receive reminder

That's enough to solve the core problem. Everything else (calendars, analytics, AI suggestions, collaboration, etc.) can come later.

A useful exercise is to describe your app in one sentence:

> "My app helps ___ achieve ___ by ___."

From this sentence, you can derive the MVP features.
