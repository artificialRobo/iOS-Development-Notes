## What is Version Control?

Version control is a system that allows you to save different versions of your code over time.

### Example Workflow

<p align="center">
    <img src="assets/versionControl.png" alt="">
</p>

1. Create a code file and write some code.
2. Save it as **Version 1** (a save point).
3. Add more code later.
4. Save it as **Version 2** (another save point).
5. If the code becomes broken or unfixable, you can:

   * Compare the current code with previous versions.
   * Revert (roll back) to an earlier version.

### Benefits of Version Control

* Tracks changes made to code over time.
* Allows comparison between different versions.
* Enables recovery from mistakes.
* Lets you return to any previous save point, not just the most recent one.

---

## Why Git?

* Git is the most popular version control system.
* Helps manage and protect code as projects grow more complex.
* Makes it easy to recover from errors and experiment safely.
