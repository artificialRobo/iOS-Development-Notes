# Notes: Cloud Firestore

## Overview

* **Cloud Firestore** is Firebase's primary database product.
* Used to store app data, such as chat messages.
* Firebase offers two databases:

  * **Cloud Firestore** (newer, recommended)
  * **Realtime Database** (older, JSON-based database)
* Firestore provides more advanced features:

  * Better querying
  * Ordering and filtering
  * More modern database capabilities

---

## 1. Create a Firestore Database

### Steps

1. Open Firebase Console.
2. Select your project.
3. Go to **Database** → **Cloud Firestore**.
4. Click **Create Database**.

### Security Modes

#### Test Mode

* Anyone with database access can read/write.
* Valid for 30 days.
* Best for development and testing.

#### Production Mode

* No public read/write access.
* Requires security rules.

**For development:** Choose **Test Mode**.

### Database Location

* Choose a region (e.g., `us-central`).
* Location affects:

  * Latency
  * Cost
* For small projects, the default region is fine.

## 3. Initialize Firestore

### In `AppDelegate.swift`

Firebase should already be configured:

```swift
import Firebase

FirebaseApp.configure()
```

Create Firestore instance:

```swift
let db = Firestore.firestore()
```

Test it:

```swift
print(db)
```

### Expected Result

* Firestore reference prints in console.
* Confirms successful initialization.

### Troubleshooting

If it fails:

* Check CocoaPods version:

```bash
pod --version
```

* Use version **1.8.4+**
* Update if necessary:

```bash
sudo gem install cocoapods
```

## 6. Save Data to Firestore

### Access Collection

```swift
db.collection(K.FStore.collectionName)
```

Collection name:

```swift
messages
```

Stored as a constant:

```swift
K.FStore.collectionName
```

### Add Document

```swift
db.collection(K.FStore.collectionName)
    .addDocument(data: [
        K.FStore.senderField: messageSender,
        K.FStore.bodyField: messageBody
    ])
```

### Data Structure

```swift
[
    "sender": messageSender,
    "body": messageBody
]
```

Firestore stores data as **documents** inside **collections**.

## Key Concepts to Remember

### Firestore Structure

```text
Collection
   └── Document
          └── Fields
```

Example:

```text
messages
   └── autoGeneratedID
          ├── sender
          └── body
```

### Useful Methods

Initialize database:

```swift
Firestore.firestore()
```

Get collection:

```swift
db.collection("messages")
```

Add document:

```swift
addDocument(data:)
```

Get current user:

```swift
Auth.auth().currentUser?.email
```
